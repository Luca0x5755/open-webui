# 單一有效登入

同一組帳號同時只保留**最後一次登入**；一旦在新的裝置或瀏覽器登入，前一次的登入即刻失效並被登出。

---

## 運作方式

| 情境 | 結果 |
| --- | --- |
| 在新裝置登入同一帳號 | 新裝置正常使用；前一裝置的登入立即失效 |
| 前一裝置繼續操作 | 下一個動作即被導回登入頁，並提示需重新登入 |
| 前一裝置的即時連線（串流／通知） | 於新登入當下立即中斷 |

## 設定

| 環境變數 | 預設 | 說明 |
| --- | --- | --- |
| `ENABLE_SINGLE_ACTIVE_SESSION` | `True` | `True` 啟用單一有效登入；`False` 關閉，回復同帳號可多處同時登入 |

修改後重新啟動服務即生效。

## 適用範圍

| 對象 | 是否受限 |
| --- | --- |
| 一般帳號登入（網頁 / App） | 是，只保留最後一次登入 |
| API Key（`sk-` 開頭） | 否，不受影響，可持續使用 |

## 部署需求

- 不需 Redis 即可運作；已使用 Redis 的部署亦完全相容。
- 首次啟動會自動建立所需資料表，無須手動操作。

## 注意事項

- 同一帳號無法在多個裝置同時保持登入；如需多人使用，請建立各自的帳號。
- 啟用前既有的登入不會被強制登出，會在各自下次登入後才納入此規則。
